# textbringer-mouse プラグインの使い方

## インストール

プラグインは既にインストール済みです。

## 有効化

`~/.textbringer.rb` に以下の行を追加してください：

```ruby
require "textbringer_plugin"
```

## 動作確認

```bash
textbringer test.txt
```

または、既存のファイルを開いて試してください。

## 機能

### マウスクリック
- **左クリック**: カーソルをクリック位置に移動
- **ダブルクリック**: クリック位置の単語を選択
- **トリプルクリック**: 行全体を選択
- **右クリック**: クリック位置の単語を選択（ダブルクリックと同じ動作）
- **ウィンドウ分割時**: 別のウィンドウをクリックしてフォーカスを切り替え

### マウススクロール
- **ホイール上**: 画面を上にスクロール
- **ホイール下**: 画面を下にスクロール

## テスト項目

`test.txt` を開いて以下を確認してください：

### マウスクリック（基本）
- [ ] シンプルなテキストでクリックしてカーソルが移動
- [ ] タブを含む行でクリックして正確にカーソル移動
- [ ] 全角文字を含む行でクリックして正確にカーソル移動
- [ ] 長い行でクリックしてカーソル移動
- [ ] ウィンドウ分割時 (C-x 2) に別ウィンドウをクリックしてフォーカス切り替え
- [ ] 行末を超えたクリックで行末にカーソル移動

### マウスクリック（新機能）
- [ ] 単語をダブルクリックして単語全体が選択される
- [ ] 行をトリプルクリックして行全体が選択される
- [ ] 単語を右クリックして単語全体が選択される
- [ ] 選択後に `M-w` (copy-region) でコピーできる
- [ ] 選択後に `C-w` (kill-region) でカットできる

### マウススクロール
- [ ] マウスホイール上で画面が上にスクロール
- [ ] マウスホイール下で画面が下にスクロール
- [ ] バッファの先頭/末尾でスクロールしてもエラーにならない

### エッジケース
- [ ] エコーエリア (下部のコマンドライン) をクリックしても問題ない
- [ ] モードライン (各ウィンドウ下部のステータス行) をクリックしても問題ない

## 対応ターミナル

- iTerm2
- Terminal.app
- tmux内での動作
- VSCode統合ターミナル

マウスをサポートしていないターミナルでも、エラーなく起動します（マウス機能のみ無効）。

## トラブルシューティング

### マウスが反応しない
- ターミナルがマウスイベントをサポートしているか確認
- tmux使用時は、マウスモードが有効か確認: `set -g mouse on`

### カーソル位置がずれる
- タブ幅の設定を確認: `M-x set-variable tab-width`
- 全角文字の表示幅を確認

## 実装の詳細

実装の詳細は `lib/textbringer/mouse.rb` を参照してください。

### 主な機能

#### マウスイベント処理
- `Window#get_char` の拡張でマウスイベントを検出
- `Curses.mousemask` でマウスイベントを有効化
- `Curses.getmouse` でマウスイベント情報を取得

#### 座標変換
- スクリーン座標からバッファ位置への変換 (`screen_to_buffer_pos`)
- タブや全角文字の表示幅を考慮した座標計算
- ウィンドウ相対座標からバッファの絶対位置へのマッピング

#### 新機能の実装
- **ダブルクリック**: `backward_word` + `push_mark` + `forward_word` で単語選択
- **トリプルクリック**: `beginning_of_line` + `push_mark` + `end_of_line` で行選択
- **右クリック**: ダブルクリックと同じ動作（単語選択）
